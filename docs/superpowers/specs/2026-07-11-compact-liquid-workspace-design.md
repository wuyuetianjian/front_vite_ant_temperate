# 紧凑液态工作台设计

## 目标

在保留现有液态玻璃、壁纸和蓝色强调色的前提下，优化整个 `/admin` 后台框架，使高频管理操作更容易扫描、切换和执行，同时降低不必要的首屏 JavaScript 与图形渲染开销。

## 范围

- 重构管理后台的全局布局：侧边导航、顶部工作栏、内容容器和页面标题区。
- 建立紧凑的后台间距、表格、筛选和状态展示规则，并应用到框架及总览页。
- 优化前端加载与渲染：管理页路由懒加载、稳定派生数据与回调、总览页的并行统计请求及更清晰的加载状态。
- 保持现有鉴权、权限菜单、i18n、主题切换、壁纸和服务品牌配置的行为不变。

本次不改变后端接口、权限模型、业务字段或各管理页的功能流程；其余业务页会继续使用现有内容，但从新的全局容器和视觉 tokens 中受益。

## 视觉与交互设计

### 全局框架

- 侧栏维持展开 220px、折叠 64px；分组导航的层级、图标和权限过滤不变。
- 增加 52px 顶部工作栏，包含侧栏开关、当前页面上下文、主题切换和用户入口。窄屏时优先保留操作，隐藏冗余上下文。
- 内容区使用统一的最大宽度、紧凑内边距（桌面 20px，小屏 16px）和一致的面板间距（12px/16px）。
- 每个页面遵循“标题与主要动作 → 可选筛选行 → 主内容面板”的结构，避免标题、说明、筛选和内容散落在不同位置。

### 液态玻璃与密度

- 保持半透明表面、壁纸衬底、蓝色主操作和深浅主题切换。
- 将高强度模糊限制在侧栏、顶部工作栏、弹层和主要内容面板；嵌套元素改用半透明填充与边框，避免多层 `backdrop-filter`。
- 卡片、表格、筛选控件和分页采用统一的紧凑尺寸；表头、行悬停、选中和空状态仍需在两种主题下清晰可辨。
- 状态信息不只依赖颜色：保留文本、图标或标签来表达权限、风险、成功和错误。

### 总览页

- 将欢迎信息、更新时间和主要管理入口收敛为紧凑的页首。
- 保留用户、角色、权限三项核心统计，以更易扫描的统计卡呈现，并提供加载占位。
- 不引入无数据支撑的图表；总览只展示系统结构摘要和可操作入口。

## 技术设计

### 路由与加载

- 将后台子页面改为 `React.lazy` 与 `Suspense` 按路由加载；登录、安装和公共 BI 页面保持现有行为。
- 为懒加载页面提供一致的轻量加载回退，避免切换时出现空白内容区。

### 渲染与状态

- 对侧栏菜单、选中项和用户菜单等派生数据使用稳定的依赖和回调，避免布局状态变化时生成不必要的对象与函数。
- 总览页继续通过 `Promise.all` 并行获取三项统计；增加卸载保护，防止请求结束后更新已卸载页面。
- 仅在壁纸 URL 或主题实际变化时更新对应 DOM/CSS 变量；不改变现有状态存储方式。

### 样式边界

- 在现有液态玻璃 token 基础上补充后台专用的尺寸与间距变量，不引入新 CSS 框架。
- 用可复用的后台类承载框架与面板样式，逐步替换布局中的一次性内联样式；保留 Ant Design 组件和既有覆盖规则。

## 错误处理与可访问性

- 页面懒加载失败时显示可理解的回退提示，避免整个后台白屏。
- 侧栏折叠、菜单和账户入口保持可键盘操作，并补齐仅图标按钮的标签。
- 保持移动端最小 320px 宽度支持，避免顶部操作和内容面板横向溢出。

## 验证

- 执行 `npm run lint` 与 `npm run build`。
- 检查浅色、深色、侧栏展开/折叠和窄屏下的后台框架。
- 登录后验证权限过滤菜单、主题切换、登出、总览统计、路由切换及壁纸显示均保持可用。
