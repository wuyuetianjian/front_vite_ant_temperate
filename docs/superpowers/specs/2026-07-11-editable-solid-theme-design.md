# 可编辑实体主题设计

## 目标

将现有主题系统改为“玻璃风格”独占毛玻璃材质。默认、暗黑、MUI、shadcn、卡通、插画、Bootstrap 和极客预设均使用不透明的实体表面。用户可在个人资料的外观设置中编辑核心视觉令牌，配置随用户保存并在登录后恢复。

## 范围

### 预设表面规则

- `glass`：保留壁纸、半透明、模糊、高光和玻璃阴影。
- 非 `glass`：关闭壁纸层、`backdrop-filter` 和透明表面；页面、导航、卡片、表格、弹窗、下拉菜单及输入控件使用预设对应的实体背景、边框与阴影。
- 每个预设继续保留其独立的主色、圆角、深浅模式和紧凑度基线。

### 用户可编辑令牌

用户可编辑并实时预览以下值：

- 主色
- 页面背景与容器背景
- 正文文字与辅助文字颜色
- 边框颜色
- 圆角
- 紧凑 / 舒适密度
- 浅色 / 深色 / 跟随系统模式

编辑器在个人资料页的外观区域中呈现。编辑从当前预设的默认令牌开始，提供恢复预设默认值操作。切换预设时清除当前自定义覆盖并加载新预设的默认值。

### 持久化与权限

- 用户核心令牌存入现有 `theme_config` JSON 字段，并通过现有 `PUT /v1/auth/me/theme` 保存。
- 前端 Zustand 状态保存即时预览值；认证用户载入后以服务端值为准。
- 保存失败时回滚预览值并显示错误反馈。
- 管理员可配置的高级令牌（组件级尺寸、阴影、字体和完整 Ant Design ThemeConfig）不进入本次普通用户编辑器；后续放置在管理后台并由管理员权限保护。

## 实现边界

1. 在主题预设模块中定义可序列化的核心令牌与默认值，并安全解析旧的空或无效 `theme_config`。
2. `ConfigProvider` 根据预设基线和用户覆盖生成 Ant Design token；CSS 变量同时覆盖应用框架与自定义组件。
3. 将现有液态玻璃 CSS 分为玻璃专用规则与实体主题规则，确保非玻璃预设没有透明或模糊声明生效。
4. 在主题面板实现即时编辑、保存、重置与失败回滚；保留现有预设选择和数据库 API。
5. 补齐中英文文案、README 用户说明，并验证构建、主题切换和后端测试。

## 验收

- 选择任意非玻璃预设时，核心组件不使用毛玻璃效果或壁纸背景。
- 选择玻璃预设时，现有玻璃效果继续可用。
- 用户修改核心令牌后，无刷新即时生效；刷新或重新登录后仍可恢复。
- 失败保存不会让本地主题与数据库状态不一致。
- 高级主题配置不会暴露给普通用户。
